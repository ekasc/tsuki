# Impeccable Design Context

## Design Context

### Users
General manga/comic fans who want to read manga online or from local archives. Public-facing product — needs to be intuitive for newcomers, not just power users. Users range from casual readers to dedicated fans who manage their own collections. Reading is the primary use case; discovery and library management are secondary.

### Brand Personality
**Three words**: Playful, expressive, refined

The interface should feel like it was made by someone who genuinely loves manga — not a generic media app. It should have personality and visual identity without getting in the way of reading. Think "a well-curated indie product" rather than "enterprise streaming service."

### Aesthetic Direction
- Playful and expressive with clear visual identity
- Both light and dark modes equally polished
- 5 theme palettes — the theme system IS a feature, not an afterthought
- The reader UI should be immersive and distraction-free
- The library/home should feel alive and welcoming, not a sterile grid
- Anti-references: Generic streaming apps (Crunchyroll-style), corporate media players, gray-on-gray admin dashboards

### Design Principles
1. **Reading first** — Every UI decision should protect the reading experience. When in doubt, remove UI chrome.
2. **Playful but not noisy** — Personality comes from intentional moments (transitions, empty states, micro-interactions), not from decoration everywhere.
3. **Theme as expression** — The 5-palette theme system is a first-class feature. Each theme should feel like a coherent world.
4. **Progressive disclosure** — The library is approachable for newcomers; power-user features reveal through interaction.
5. **Both modes equal** — Light and dark are peers, not a primary/fallback relationship. Each must look intentional.

### Accessibility Target
WCAG AA compliance.
